# 登录鉴权方案说明

## 一、使用什么

本项目登录鉴权使用的是 Spring Security 加 JWT（JSON Web Token）。

Spring Security 是 Spring 生态的安全框架，负责拦截请求、校验权限、处理登录认证。JWT 是一种轻量级的令牌格式，用于在前后端之间安全传递用户身份信息。

## 二、JWT 的原理

用户登录成功后，服务端生成一个 JWT 令牌返回给前端。这个令牌包含三部分：头部（算法类型）、载荷（用户 ID、用户名、角色等自定义信息）、签名（用密钥对前两部分加密生成的防篡改校验值）。

前端收到令牌后存储在浏览器本地，后续每次请求都在请求头中带上这个令牌。服务端收到请求后，用同样的密钥验证签名是否有效，如果令牌被篡改过签名就对不上，直接拒绝。验证通过后从载荷中取出用户信息，知道是谁在操作、有什么权限。

JWT 本身是明文的，任何人拿到令牌都能看到里面的内容，但因为签名需要用密钥来生成和验证，所以没人能篡改。因此密钥必须保密，泄露后任何人都可以伪造令牌。

JWT 有过期时间，本项目中设为二十四小时，过期后前端需要重新登录获取新令牌。

## 三、Spring Security 的作用

Spring Security 在本项目中承担三个职责。第一是密码加密，用户注册或修改密码时，密码通过 BCrypt 算法加密后存入数据库，登录时用加密后的密文和用户输入的明文比对，数据库里存的不是明文密码。第二是拦截请求，除登录接口外所有请求都需要携带有效 JWT 令牌，没有令牌或令牌过期直接返回未授权错误。第三是权限控制，根据 JWT 载荷中的角色信息判断当前用户是否有权限执行某个操作，比如只有管理员能删除数据，教师和学生不能。
