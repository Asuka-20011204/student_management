# Git 克隆项目流程

## 一、克隆是什么

克隆（clone）是把远程仓库的完整代码下载到本地的操作。克隆后本地会生成一个和远程仓库一模一样的文件夹，包含所有代码文件和历史提交记录，并且自动关联好远程仓库地址，后续可以直接 pull 和 push。

## 二、操作步骤

第一步，获取仓库地址。在 GitHub、GitLab 或 Gitee 的项目主页点击"Code"按钮，复制 HTTPS 或 SSH 地址。HTTPS 地址格式是 https://域名/用户名/仓库名.git，克隆时需要输入账号密码。SSH 地址格式是 git@域名:用户名/仓库名.git，需要提前配置好 SSH 密钥，好处是后续操作不用反复输密码。

第二步，打开终端进入想要存放项目的目录，执行克隆命令，命令格式为 git clone 后面跟上仓库地址。执行后会看到一个进度提示，显示正在接收对象。

第三步，等待下载完成，当前目录下会生成一个以仓库名命名的文件夹，进入这个文件夹即可开始开发。

## 三、克隆后的必要操作

克隆完成后，进入项目目录，先确认当前分支是否正确，通常开发用的是 dev 或 develop 分支而不是 main 或 master。如果远程有 dev 分支但本地默认在 main 分支，需要执行切换分支命令并关联远程分支，命令是 git checkout -b dev origin/dev，这行命令会在本地创建 dev 分支并关联远程的 dev 分支。

然后安装项目依赖。如果是 Maven 项目执行 mvn install 或让 IDEA 自动下载依赖。如果是前端项目执行 npm install 或 yarn。依赖安装完成后即可运行项目开始开发。
