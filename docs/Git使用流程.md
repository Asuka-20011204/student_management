# Git 使用流程

## 一、正确顺序

Git 提交代码的正确顺序是：先拉取（pull），再提交（commit），最后推送（push）。

先拉取是为了把远程仓库最新的代码同步到本地。如果别人在你之前推送了代码，你不先拉取就直接推送会被拒绝，因为本地版本落后于远程。拉取后如果有冲突，在本地解决冲突再提交，避免污染远程仓库。

提交是把本地修改保存到本地仓库的历史记录中。提交只发生在本地，不会影响远程，别人此时看不到你的提交。

推送是把本地仓库的提交记录上传到远程仓库。推送后，团队其他人就能拉取到你的代码了。

总结流程：pull 拉取最新代码 → commit 本地提交 → push 推送到远程。

## 二、提交时必须填写的内容

提交时必须填写提交信息（commit message），格式为：类型冒号空格简短描述。

类型用于说明本次提交的性质，常用类型有：feat 表示新功能、fix 表示修复 bug、refactor 表示重构代码、docs 表示文档变更、test 表示测试相关、chore 表示杂项任务。描述用简洁的语言说明做了什么，中文英文均可，但要让人一眼看懂。

提交信息的正文部分是可选的。如果改动较复杂，可以在简短描述下面空一行，再写详细的说明文字，解释为什么要这样改、影响了哪些模块。
